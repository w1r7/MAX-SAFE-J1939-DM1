DM1 - Active Diagnostic Message

How DM1 works - expand for explanation

DM1 is the active fault message. It uses PGN 65226 / 0x00FECA.

SPN identifies what signal or component has the fault. FMI identifies how it failed. OC is the occurrence count.

For this customer integration, the calculator uses single-packet DM1 only. If several faults are active, the shared trace rotates through them one-by-one. No DM1 BAM / multi-packet message is generated.

How to use this view

Choose a mapped fault from the list. The tool shows the DM1 message that MAX-SAFE would transmit.

Software ID - PGN 65242 / SOFT

  

    How Software ID works - expand for explanation
    
Enter the MAX-SAFE software number, for example MSB-5.39.89. In standard mode, the tool builds the J1939 Software Identification message.

    
PGN 65242 / 0x00FEDA is an on-request message, not a cyclic message. The Software button in the common trace area button simulates an external request for PGN 65242. MAX-SAFE then sends the software identification response one time only.

    
Standard mode uses Byte 1 as the number of software identification fields, then sends the software text as ASCII and terminates it with *. If the software text is 6 characters or fewer, the result fits in one 8-byte CAN frame. Longer text is sent using J1939 Transport Protocol BAM + TP.DT frames.

    
The software response shares the same trace buffer as DM1. If the DM1 trace is already running at 1000 ms, clicking the request button inserts the software response into the trace at the click time, between the DM1 fault lines.

Common CAN Trace

  

    How the shared trace works - expand for explanation
    
The trace area represents one shared CAN bus log for the whole calculator. Every tab and every future message type writes to the same trace buffer and the same export file.

    
Start begins the cyclic trace source currently implemented in the calculator. At this stage, that cyclic source is DM1. The Start button is disabled while the trace is already running.

    
Stop stops the cyclic trace source. It is only active while the trace is running. Reset clears the trace buffer and remains available all the time.

    
Customer requests > Software simulates a request for PGN 65242 and appends the Software Identification response once. If DM1 is running every 1000 ms, the SOFT response is inserted between the DM1 frames at the click time.
